# 图片比较与重命名工具

这是一个使用 Python 的 Tkinter 库构建的图形用户界面（GUI）应用程序。它旨在帮助用户直观地比较本地图片与远程生成的图片，并根据比较结果对本地文件进行重命名。

## 功能特性

- **基于文件夹的图片加载**：选择一个主目录，工具将自动查找并让您从以 "page_" 开头的子文件夹中进行选择。
- **并排图片比较**：顶部显示一张大的参考图片，中间是待检查的本地图片，右侧是远程生成的图片，方便进行比较。
- **高效重命名**：只需单击一下，即可根据图片是否正确来重命名本地文件。
- **自动处理下一张**：处理完当前图片后，程序会自动加载下一张图片。
- **坐标系功能**：允许指定列和行，并将这些信息嵌入到新的文件名中。

## 运行环境

请先运行一下指令配置环境（建议使用python-3.13版本）

```
pip install -r requirements.txt
```

## 使用说明

1. **启动程序**：运行 Main2.py 脚本。
2. **选择主文件夹**：点击“浏览”按钮，然后导航到您的主目录（例如 E:/ID-Name）。
3. **选择页面**：选择主文件夹后，“Page文件夹”下拉菜单将填充所有找到的 page_ 文件夹。选择您想要处理的页面。
4. **图片显示**：
   - 大的参考图片（例如 page_1.png）将出现在顶部面板中。
   - 第一张待检查的本地图片（例如 page_1 文件夹中的 1.png）将出现在“待检查图片”面板中。
   - 根据 .txt 文件中相应文本生成的远程图片将出现在“正确图片”面板中。
5. **设置坐标**：
   - 使用“列”和“行”输入框来指定当前图片的坐标。
   - 点击“增加列”将列数加一，并将行数重置为 1。
   - 点击“增加行”将行数加一。在您将图片分类为“正确”或“错误”后，行数也会自动增加。
6. **比较与分类**：
   - 如果“待检查图片”与“正确图片”匹配，请点击“正确”按钮。
   - 如果它们不匹配，请点击“错误”按钮。
   - 如果本地图片是多余的，可以点击“删除”按钮将其移除。
7. **继续**：做出选择后，当前图片将被重命名（或删除），然后程序会自动加载序列中的下一张图片。
8. **完成**：当所选页面文件夹中的所有图片都处理完毕后，将显示“任务完成”的消息。

## 文件命名规则

当您对图片进行分类时，它将根据以下格式进行重命名：

{状态}_{页面}_{列}_{行}_{文本}.png

- **{状态}**：0 代表正确，1 代表错误。
- **{页面}**：页面的编号（例如，从 page_1 中提取的 1）。
- **{列}**：来自输入框的列号。
- **{行}**：来自输入框的行号。
- **{文本}**：来自 .txt 文件中的相应文本。

**示例：**
如果您正在处理 page_1，列号为 3，行号为 5，图片对应的文本是 "apple"，并且您点击了“正确”按钮，则原始文件将被重命名为：

0_1_3_5_apple.png
